# 순간 수집 앱 리디자인 — 기획 컨셉

> 이 문서는 사람 개발자를 위한 한글 기획/컨셉 문서입니다. 에이전트용 문서가 아니므로 `AGENTS.md` 색인에 포함하지 않습니다.
> 여기 적힌 것은 **아직 구현되지 않은 방향**입니다. 실제로 구현되면 코드 소유 계층과 동작 정의는 `docs/features/`에 영문 기능 문서로 별도 작성합니다(기능 문서는 "구현된 동작만 기술" 규칙이라 컨셉 단계에서는 만들지 않습니다).
> 최초 논의: 2026-07-23.

---

## 이 문서의 목적

현재 Snaply의 UI는 흔한 촬영/편집 앱 구조로, 차별점이 약합니다. 이 문서는 앱을 **"순간을 수집하는 앱"**으로 재정의하는 리디자인 방향을 기록합니다. 제품 프레이밍, 데이터 모델, 시각 언어, 화면 설계, 모션까지 지금까지 확정한 결정과 아직 열려 있는 질문을 한곳에 모읍니다.

함께 만든 시각 프로토타입(HTML)은 같은 폴더에 있습니다. 브라우저로 열어보세요.

- [`visual-language.html`](visual-language.html) — 팔레트·타이포·모티프 스타일 가이드
- [`screens.html`](screens.html) — 네비게이션 구조와 담기/현상/수집 세 플로우 화면 목업
- [`motion.html`](motion.html) — 눌러보는 꾹-담기, 재생되는 현상 세리머니 (인터랙티브)

## 1. 제품 프레이밍

핵심 전환은 **"영상을 만든다"에서 "순간을 모은다"로 무게중심을 옮기는 것**입니다.

- 목표: 하루 또는 여행의 순간순간(3~5초)을 기록하고, 이를 묶어 숏폼 영상을 쉽게 만들며, AI 편집까지 지원한다.
- 차별화 축: 촬영/편집 도구(레드오션)가 아니라 **수집 경험**. 관심을 끄는 두 축은 (1) 독특한 디자인, (2) 수집욕을 부르는 구조.
- "AI로 찍어낸 흔한 앱" 느낌을 벗어나는 것이 리디자인의 1차 목표입니다.

---

## 2. 핵심 메타포 — 필름 / 현상

순간에 **물성**을 부여하기 위해 필름 사진의 실제 작업 세계에서 은유를 가져옵니다.

| 개념 | 은유 |
| --- | --- |
| 컷(3~5초) | 네거티브 필름 한 컷 |
| 롤/프로젝트 | 인화한 앨범·릴 |
| 하루 / 여행 | 필름 한 롤 |
| AI 편집 | 현상소(Darkroom) |

이 은유에서 **"지연된 현상"**이라는 시그니처 훅이 나옵니다(§4 참조).

---

## 3. 데이터 모델 (확정)

세 가지를 분리합니다. 컷은 원본, 롤은 그 원본을 골라 엮은 편집본이며, 관계는 **N:M**입니다.

```text
컷(Clip) ── N:M ── 롤(Roll / 프로젝트)
  │                     │
  · 3~5초 원본           · 성격(타입): 외형·톤·기본 길이
  · 시간/위치/태그        · 수집 규칙: 어떻게 컷이 들어오는가
  · 보관함에 1개만 존재    · 컷별 편집정보(트림·순서·속도)를 "롤이" 소유
  · 불변(네거티브)         · 현상 → 숏폼 영상 결과물
```

- **컷은 보관함(아카이브)에 한 번만 존재**하고, 롤은 컷을 **참조**만 합니다. 한 컷이 여러 롤에 동시에 들어갈 수 있습니다(예: 노을 컷 하나가 데일리 롤·여행 롤·"노을 모음" 롤에 모두).
- **원본 컷은 불변**입니다. 트림·순서·속도·필터 같은 편집 정보는 **롤이 각자** 들고 있습니다. 같은 컷이 롤A에선 3초, 롤B에선 5초로 살 수 있습니다.

### 롤이 따로 갖는 세 설정

| 설정 | 뜻 | 예 |
| --- | --- | --- |
| (A) 성격/타입 | 롤이 무엇이냐. 외형·BGM 톤·기본 길이·커버 스타일 결정 | `일상` · `여행` · `챌린지` · `성장기록` · `자유` |
| (B) 수집 규칙 | 컷이 이 롤에 **어떻게** 들어오는가 | 아래 표 |
| (C) 지향 화면비 | 롤이 지향하는 방향·비율. 결과 릴의 화면비를 결정하고 담기를 유도 | `세로 9:16` · `와이드 16:9` · `정방 1:1` |

### 수집 규칙

| 규칙 | 동작 | 대표 프리셋 |
| --- | --- | --- |
| 수동 | 내가 직접 골라 담음 | 자유 |
| 기간 | 지정한 날짜 범위의 컷 자동 수집 | 여행 |
| 위치 | 지오펜스 안에서 찍은 컷 자동 수집 | 여행 |
| 태그 | 특정 태그 컷 자동 수집 | 커스텀 |
| 그날 전체 | 오늘 담은 컷 전부 | 데일리 |

> "데일리 롤"과 "여행 롤"은 별도 기능이 아니라 이 규칙의 **프리셋 조합**일 뿐입니다.
> - 데일리 롤 = `성격:일상` + `규칙:그날 전체` (매일 자동 생성)
> - 여행 롤 = `성격:여행` + `규칙:기간 + 위치`

**위치 규칙은 이미 구현된 지오펜싱(`geofence-monitor`)을 재활용**합니다. 새 기능이 아니라 기존 투자 회수입니다.

### 지향 화면비 (방향)

필름엔 원래 화면비가 있습니다(표준 세로, 와이드=시네마스코프). 이를 롤에 주입하는 특성으로 둡니다.

- **세로 롤** (9:16) — 기본. 숏폼 문법.
- **와이드 롤 / 시네마스코프** (16:9) — 시네마틱. "여행 시네마", 공연, 풍경처럼 **의도적으로** 고르는 성격.
- (옵션) **정방** (1:1).

방향은 **강제가 아니라 "가급적" 유도(소프트 가이드)**입니다.

| 상황 | 동작 |
| --- | --- |
| 와이드 롤에 담으려는데 폰이 세로 | 뷰파인더에 가로 프레임 가이드 + 은은한 `↻ 가로로` 힌트, 링 옆 방향 인디케이터 |
| 가로로 돌리면 | 가이드 확정(햅틱·색 변화) |
| 그냥 세로로 담으면 | **막지 않음.** 저장은 되지만 컷에 `방향 불일치` 마커, 컨택트시트에서 레터박스로 표시 |

롤의 지향 화면비는 **결과 릴의 화면비**를 결정합니다. 담기 마찰은 0으로 유지하되, "이 롤은 가로다"를 몸으로 알립니다.

### N:M에서 확정한 규칙

| 항목 | 결정 |
| --- | --- |
| 삭제 | `롤에서 빼기`(참조만 제거, 원본 안전)와 `보관함에서 삭제`(영구, 모든 롤에서 사라짐 — 경고 필요)를 UI에서 분리 |
| 편집 소유권 | 원본 컷은 불변. 트림·순서·속도는 롤이 소유 |
| 방향 불일치 | 컷은 불변 원본이라 여러 방향의 롤에 섞일 수 있음. 화면비가 안 맞는 컷은 릴에서 **레터박스(추천, 필름 게이트 테두리 연출)** / 블러 확장 / 스마트 크롭 중 롤 기본값으로 채움. 원본은 왜곡하지 않음 |
| 복잡도 | 성격·규칙·화면비를 날것으로 노출하지 않음. 화면엔 프리셋 카드만, 풀 커스텀은 한 겹 뒤 |

## 4. 수집욕을 부르는 장치

| 장치 | 내용 |
| --- | --- |
| **지연된 현상** | 담는 순간엔 완성본을 주지 않는다. 하루/여행이 끝나면 "현상하기"가 뜨고 AI가 엮은 릴이 **처음** 공개된다. 하루 종일 기대를 쌓게 하는 핵심 리텐션 훅 |
| **빈칸을 보여준다** | "오늘의 롤 3/12컷", 선반의 빈 롤 슬롯 — 채워질 자리가 보이면 채우고 싶어진다 |
| **테마 롤 / 시즌 컬렉션** | 롤에 정체성 부여. 다 모으면 특별 커버·뱃지 |
| **스트릭** | 연속 기록은 넣되 흔한 불꽃 아이콘 대신 필름 각인·도장 느낌 |

**AI의 위치**: 편집 툴을 전면에 세우지 않는다. 기본은 자동으로 엮어주는 "현상소"로 숨기고, 심화는 "다르게 현상"(무드·길이·BGM) 뒤에 둔다. 타임라인 편집기는 원하는 사람만 진입.

## 5. 시각 언어 (확정) — 암실 / 라이트박스

빈티지(크림색+세리프+세피아)는 요즘 AI가 찍어내는 필름 클리셰라 **의도적으로 피합니다.** 대신 필름의 실제 작업 공간에서 색을 가져옵니다.

- **다크 테마 고정.** 앱은 하나의 세계 = **암실(Darkroom)**. 라이트 테마는 없습니다.
- **라이트박스(냉광 시안)는 테마가 아니라 "현상하는 순간"의 연출**로만 등장합니다. 어둠→빛으로 이미지가 떠오르는 그 순간.
- 색의 시그니처는 단색 포인트가 아니라 **온도 대비** — 따뜻한 앰버(세이프라이트·필름 베이스) vs 차가운 시안(라이트박스).

### 팔레트

| 토큰 | HEX | 역할 |
| --- | --- | --- |
| Ground | `#16110D` | 암실 바닥. 순수 흑색이 아닌 따뜻한 흑갈색 |
| Ember | `#EA5E38` | 세이프라이트. 주 강조색·촬영 버튼 |
| Amber | `#E7A24A` | 네거티브 필름 베이스. 엣지 코드 각인 |
| Lumen | `#82D6CE` | 라이트박스 냉광. 차가운 대비축 |
| Ink | `#F1E6DA` | 따뜻한 오프화이트 텍스트 |

### 타이포그래피

- 표제: 굵은 휴머니스트 산세리프(800, 음수 자간).
- 정체성: **모노스페이스 엣지 프린트** — 필름 가장자리에 빛으로 찍힌 각인을 UI 언어로. 예: `ROLL 019 · 2026-07-23 · 03/12 · ISO 400`.

### 모티프

퍼포레이션 필름 프레임(현상완료/미현상/현상중 스캔), 꾹-담기 링, 수집 선반+빈 슬롯, 은은한 그레인(~6%).

## 6. 화면 설계 (확정)

### 네비게이션

하단 2탭 + 가운데 세이프라이트 촬영 버튼. 담기는 언제나 한 번의 탭. 설정·프로필은 보관함 구석에.

```text
[ 오늘 ]      ( 담기 )      [ 보관함 ]
  홈       중앙·상시 촬영     선반+아카이브
```

### 세 플로우

| 플로우 | 흐름 |
| --- | --- |
| **A. 담기** | 뷰파인더(담을 롤 확인) → 꾹 누르는 5초 링 → 손 떼면 프레임이 롤로 빨려듬, 카운터 상승. 완성본은 안 보여줌(미현상) |
| **B. 현상** | 롤 상세(격자 컨택트시트, 미현상 컷 + 빈 슬롯) → `현상하기` → 스캔 라인 세리머니 → 릴 첫 공개 → `다르게 현상` |
| **C. 수집** | 새 롤 프리셋 카드(오늘/여행/챌린지/자유) → 보관함 선반(롤 커버 + 빈 슬롯) |

- 뷰파인더는 라이트모드가 없으므로 **항상 다크(암실)**.
- 담을 롤이 와이드 지향이면 뷰파인더가 가로 프레임 가이드로 유도합니다(§3 지향 화면비). 강제가 아닌 소프트 가이드이며, 방향이 안 맞아도 담기는 됩니다.
- 롤 상세는 **격자 컨택트시트**로 확정(가로 필름스트립 아님).

## 7. 모션 (확정 방향)

| 모션 | 타이밍 · 이징 |
| --- | --- |
| 담기 링 채움 | `0→5s · linear` (stroke-dashoffset) |
| 릴리즈 스냅 | `250ms · ease-out-back` — 프레임 fly-to-strip + medium 햅틱 |
| 현상 스캔 | `2000ms · ease-in-out` — 냉광 스윕 top→bottom |
| 컬러 블룸 | `900ms · stagger 380ms` — 그레인 dissolve → 색 |
| 릴 조립·공개 | `500ms · cubic-bezier` — scale-in reveal |
| 저감 모션 | `prefers-reduced-motion` 시 스캔·블룸 생략, 최종 상태 즉시 |

**현상 세리머니가 앱의 하이라이트**입니다. 지연된 기대를 터뜨리는 감정적 보상 지점이므로 풀스크린 연출로 다룹니다. 동작하는 예시는 [`motion.html`](motion.html) 참조.

열린 질문: 담기 링 5초가 적절한지(3초 후보), 현상 세리머니 총 길이(현재 ~3.5초).

## 8. 현재 코드와의 관계

이 리디자인은 현재 구현된 프로토타입 흐름을 대체·발전시킵니다.

- 현재 `capture-flow`(무드/길이 선택 → 촬영 → 시뮬레이션 편집 → 결과)와 `home`, `recording-archive`는 각각 `Partial`/`Prototype` 상태입니다(`docs/features/README.md`).
- 리디자인 후: 촬영은 "꾹-담기"로, 편집 결과 화면은 "현상 세리머니 → 릴"로, 아카이브는 "보관함(컷) + 선반(롤)"으로 재구성됩니다.
- **재활용**: 위치 규칙 = 기존 `geofence-monitor`, 로컬 영상 저장 = 기존 `manage-recordings`/`recording-files`.
- FSD 관점의 새 엔티티 후보: `clip`(컷), `roll`(롤), 수집 규칙 엔진. 실제 슬라이스 배치는 구현 착수 시 `docs/architecture`·`docs/conventions` 규칙에 따라 결정합니다.

## 9. 아직 미결정 / 다음 단계

- MVP 범위: 데일리 롤만으로 먼저 낼지, 테마 롤까지 포함할지.
- 데이터 모델을 처음부터 N:M + 규칙 기반으로 설계(권장)하되, v1 UI에 노출할 프리셋 확정.
- 담기 링 시간, 현상 세리머니 길이의 실제 튜닝.
- 프리셋별 기본 지향 화면비(예: 여행=와이드로 둘지)와 방향 불일치 기본 채움 방식(레터박스 기본 제안).
- 구현 착수 계획(슬라이스 설계, 마일스톤).
